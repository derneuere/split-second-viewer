FMOD Events (.fev)

An .fev file is an FMOD Designer event project — the metadata side of the audio system that the in-house LLAudio2 EventManager loads at runtime. It defines event groups, categories, mixer parameters and the wavebank references that point into the matching .fsb sample bank. Every .fev in Split/Second is the FEV1 revision emitted by FMOD Ex 4.28's Designer tool.

Role & pairing

The .fev never contains audio samples; it only describes events (named, parameterised sounds) and the categories that mix them. At load the game opens the project, then opens the sibling .fsb it was compiled against. The FMOD error string baked into the eboot — "FSB mismatches the FEV it was compiled with…" — confirms the two files are version-locked. Gameplay then triggers events by name; the eboot exposes plenty of these (fmod_event_name, trigger_fmod_event, Menu/Menu/hud_line_pass_lap_03, Surface/Landing/LandImpact, Tonic/Tonic/Engine).

Some events resolve to a path like Music/MusicLooped.fsb.fev (a doubled extension visible in the eboot), i.e. the event name itself carries the bank reference. Reverb/group routing lives in the dedicated Global/ReverbAndGroups.fev project.

File header (FEV1)

The header is short and was decoded by diffing the tiny ReverbAndGroups.fev against Menu.fev. The 4-byte version tag is constant; the project name is the first length-prefixed string.

OffsetLengthTypeNameDescription
0x000x4char[4]magicAlways "FEV1" (46 45 56 31).
0x040x4uint32_tversionFMOD Designer FEV version. Constant 00 00 3a 00 across all 73 projects (verified by dumping the first 8 bytes of every .fev; revision 0x3A, matching the 4.28 toolchain).
0x080x4uint32_tbankHeaderSize / offsetSize word preceding the wavebank list (e.g. 0x04b0 in Menu, 0x24 in ReverbAndGroups).
0x0C0x4uint32_tfield_0CSecond size/offset word (purpose not fully pinned; tracks header growth).
0x100x4uint32_t (LE)nameLenLength of the project name including the NUL terminator. 0x05 for "Menu", 0x10 for "ReverbAndGroups".
0x14nameLenchar[]projectNameProject name, NUL-terminated. Matches the file's base name.
…var—wavebank listCount + length-prefixed wavebank names (the FSBs this project references).
…var—category treeLength-prefixed category names + per-category float volume (1.0 default) and routing.
…var—event groups / eventsNested event groups, each holding events with GUIDs, parameter ranges and layers.

String encoding

Every string in an FEV is stored as a 32-bit little-endian length (counting the NUL) followed by the bytes and the terminator. This is why "master" appears as 07 00 00 00 6d 61 73 74 65 72 00 (len 7 = 6 chars + NUL). Recognising this pattern makes the whole tree walkable even before every field is typed.

Category / group tree

After the project name comes the category list — these are Split/Second's audio mixer buses, each carrying a default volume of 1.0f (00 00 80 3f). The full bus list lives in Global/ReverbAndGroups.fev; individual project FEVs carry only the subset of buses they actually use (e.g. Menu.fev contains just master, music, HUD and _Template). The complete in-file order in ReverbAndGroups.fev is:

"Menu" is a project/event-group name whose events route to the HUD bus; surface-scrape sounds sit under Surfaces. The bus names above were extracted directly from the file's string table.

What is still open

The header is well understood and the category/wavebank tree is readable, but the deeper event records (per-event parameter curves, layer/sound-def graphs, DSP chains) are only partially mapped. The _Template block is followed by a named 3D/reverb property schema — the string table exposes the field names directly: LowPassMix, ReverbMix, InnerRadius, OuterRadius, VolumeFalloffResponse, VolumeAtInnerRadius/OuterRadius, LowPassFalloffResponse, ReverbFalloffResponse, ReverbWetMixAtInnerRadius/OuterRadius, OmniAtInnerRadius/OuterRadius, the cone group (ConeAngle, ConeInnerAngle, ConeOuterAngle, ConeInnerVolume, ConeOuterVolume, ConeInnerLowPass, ConeOuterLowPass), WorldHeadBlend, HeadPositionX/Y, DopplerShiftRange, ProjectionDistance, and the flags Force2dPanning, ForceOriginPanning, ForceHeadShot, AllowVirtual. These are the per-event 3D positioning/rolloff parameters; their binary value layout (vs. just the name strings) is what is still to be pinned down, along with full event-graph decoding.

How we know

  • Magic / version: FEV1 + 00 00 3a 00 are byte-identical across Menu, Music, Speech, Tonic, Collision, Hypercar_03 and ReverbAndGroups.
  • String format: the (len, bytes, NUL) pattern reproduces exactly for "Menu" (len 5), "ReverbAndGroups" (len 16) and every category name.
  • Category meaning: bus names like PlayerEngine, HyperReal, Powerplays match the game's audio subsystems and the eboot's DynamicMixer.xml / PatchBay.xml references.
  • Pairing: the eboot's FMOD "FSB mismatches the FEV" error proves the version-locked FEV↔FSB relationship.
