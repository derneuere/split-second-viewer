AI (SSAI)

SSAI is Black Rock Studio's in-house racing-AI system for Split/Second. Internally the opponent driver is an AIController that reads its handling behaviour from LDC tables (per-speed steering, braking, look-ahead and drift parameters), gains a power bar through drafting, drifting, fast driving and jumping, and is dressed up by the AIPilotsDatabase roster of named pilots, cars and colours. Most of the tuning lives in plaintext .params files under Deferred/SSAI/ and Deferred/GameLogic/, so the values below are quoted directly from the shipped devkit data.

The driving model: LDC tables

An AI car does not have a single fixed handling profile. Instead each archetype has an LDC file (the engine and data only ever use the acronym; "driving constants" is the plain reading, but no expansion appears in the eboot) that stores a full set of driving parameters at three reference speeds — 60, 120 and 180 — and the controller interpolates between them based on the car's current speed. The speed ladder itself is defined globally in GameplayParameters.params: 

/GameplayParameters/LDC:
	'000;Speed Intervals' = 60.0 (1.0, 300.0);
	'001;Initial Speed'   = 60.0 (1.0, 300.0);
	'002;Final Speed'     = 180.0 (1.0, 300.0);

So the three Speed 60 / 120 / 180 sections you see in every *_LDC.params file are exactly Initial Speed → Final Speed in Speed Intervals steps. Each section holds the same 20 numbered variables. The EBOOT confirms the load order: SSAI/Default_LDC.params, then SSAI/%s_LDC.params per class, then a track-specific override SSAI/%s/%s_LDC.params (e.g. docks_A/AISupercar_01_LDC.params).

A trailing per-file section records a baseline lap time used by the test harness, e.g. /AI/LDC/Default → '000;Lap time' = 76.399773. These 20 variables can be individually toggled at race-setup time; AISkillTests.params exposes 'Enable Variable 0' … 'Enable Variable 15' plus a Friction Speed Loss Multiplier and IsLearning / IsRecording flags that drive the evolution/recording mode.

The "hazard" multipliers (010–015) let the LDC dial back look-ahead and target speed when an AI is approaching an active Powerplay hazard, graded by hazard difficulty (Easy/Medium/Hard). In the shipped Default and per-car tables they are all left at 1.0 — i.e. AI does not slow for hazards by default — but the range (down to 0.5 speed / 0.2 look-ahead) shows the intended behaviour.

LDC archetypes and overrides

Deferred/SSAI/ holds one base table per class plus numbered variants. The base files are AISupercar_LDC.params, AIMusclecar_LDC.params, AITruck_LDC.params and NemesisTruck_LDC.params, each ~4.2 KB. The numbered variants (AISupercar_01_LDC.params … _05, etc.) and the per-track subfolders (docks_A/, docks_B/, airport_test_03_A/, FE_car_unlock_A/) are evolved/recorded refinements — typically with the same structure but non-round, machine-tuned numbers. Compare Default vs the evolved AISupercar_LDC, where look-ahead distances have drifted to oddly precise values:

AI power bar gain rules

AI opponents charge a Powerplay power bar exactly like the player, but the rate at which they earn it from each activity is set centrally in GameLogic/GameplayParameters.params under AI Powerbar:

ActivityRuleValue
DraftingMin distance to start gaining5.0 m
DraftingMax angle difference to player5.0°
DraftingUnits gained / sec4.0
DriftingUnits gained / sec14.0
Drive Fast% of max speed to start gaining0.6
Drive FastUnits gained / sec1.0
JumpMin air time to start gaining0.5 s
JumpUnits gained / sec10.0

Drifting is by far the richest source (14 units/s) and fast cruising the weakest (1 unit/s), which is why AI cars favour aggressive lines into corners. These base rates are then scaled by a per-difficulty AI powerbar gain multiplier (start-of-race, end-of-race, and per-skill — see Difficulty & Race Director). The same file also sets the AI skill-update cadence: 

/GameplayParameters/AI Skills:
	'000;Skill update delay'           = 0.0   (0.0, 100.0);
	'001;Skill update buffer distance' = 200.0 (0.0, 1000.0);

AIPilotsDatabase — the opponent roster

GameLogic/AIPilotsDatabase/AIPilotsDatabase.params defines the pool of opponent "pilots" the game draws from. It contains 98 entries (AI_Driver_No_01 … AI_Driver_No_98), each with a VehicleDesc (vehicle name, livery RGBA, racing number, and 15 achievement-level fields) and a top-level record giving its Name and DriverSkill. 

/AIPilotsDatabase/AI_Driver_No_01/VehicleDesc:
	'000;Vehicle Name' = Unique_04;
	'005;Number' = 1 (0, 99);
/AIPilotsDatabase/AI_Driver_No_01:
	DriverSkill = 1.0 (0.0, 1.0);
	Name = WEIGAND;

There are exactly seven driver surnames, each appearing 14 times across the 98 slots:

Driver names (×14 each)
WEIGAND · DECKMAN · BAUER · BROKER · YARDLEY · WOLTZ · KULINSKI

The roster is laid out in skill bands. DriverSkill takes five discrete values, and the database walks from the top band downward as the driver number increases:

DriverSkillEntriesNotes
1.020Top tier (drivers 01–20)
0.820Drivers 21–40
0.620Mid pack
0.420Lower pack
0.218Bottom tier

The vehicle column is spread across the full roster — the Unique_01…04 "hero" cars and Musclecar_09/Supercar_08 sit in the 1.0 band, while trucks (Truck_01) and the muscle/super grinders cluster at lower skills. Liveries are mostly black (0,0,0) in this devkit data; the Number field simply equals the driver index.

Skill vs. LDC. DriverSkill is a 0–1 scalar that blends between two extremes — the LDC handling table at skill 1, and a degraded version at skill 0 governed by the Skill == 0 parameters in the difficulty files (extra speed multiplier, reduced look-ahead, wider racing-line offset). The pilot's DriverSkill sets where that car sits on the blend before the per-race difficulty/race-director logic pushes it around.

EvolutionConfig — the automated fitness-test harness

SSAI/EvolutionConfig.params is the configuration for an offline AI auto-tuning harness. Rather than hand-balancing the LDC tables, designers could let the build sweep a driving variable across a range, run the AI through a battery of point-to-point "tests", and keep whatever value produced the best result. The EBOOT confirms the loop: SSAI/Evolution/Enabled, SSAI/Evolution/Debug Render, and results written to SSAI/EvolutionResults_%s.txt.

The config is organised as EvolutionConfig/<Category>/<Variable>/Test0…Test7. In the shipped data the category is Drift and the two swept variables match the PerCarInfo drift tweaks: Curvature ahead factor and Sideways factor. Each variable defines up to 8 test waypoints (Test0–Test7), of which the first four are Valid = true and the rest disabled: 

/SSAI/EvolutionConfig/Drift/Curvature ahead factor/Test0:
	'000;Valid' = true;
	'001;Initial position X' = -550.0;
	'002;Initial position Y' = -0.6;
	'003;Initial position Z' = -600.0;
	'007;Respot speed proportion (to the max speed of the car)' = 0.75;
	'008;Final zone center X' = -409.20001;
	'009;Final zone center Y' = 0.6;
	'010;Final zone center Z' = -501.39999;
	'011;Final zone radius' = 15.0 (0.0, 20.0);
	'012;Maximum time allowed for test (in secs)' = 15.0 (0.0, 30.0);

Each test respots the AI car at an Initial position at a fraction of its top speed (the Respot speed proportion, 0.75–0.85 in shipped tests), then times how long it takes to reach a spherical Final zone (centre + radius) before the per-test timeout (10–20 s). A car that lands inside the zone in time "passes"; the harness scores the swept value by aggregate pass/time across all valid tests. The two control fields per variable are:

FieldValueMeaning
Allow drifting when testing this variabletruePermit drift during the run
Number of variables to try in the range for this variable11 (2–20)How many samples across the range

So for each of the two drift factors the harness samples 11 candidate values, runs the four valid point-to-point tests for each, and writes a ranked EvolutionResults_*.txt. This is the same family of automated test rigs as Megabowles (cMegabowlesController) for Powerplays — Black Rock leaned heavily on data-driven QA harnesses.

Where Evolution tunes handling, the related AISkillTests game mode (GameLogic/GameModes/AISkillTests.params) plays a full race with the LDC variables selectively enabled and IsRecording/IsLearning flags set, logging to SSAI/AISkillTests_%d_…%d.log and SSAI/LDC_%d_…%d.log. Together they form the LDC authoring pipeline that produced the evolved per-car tables.
