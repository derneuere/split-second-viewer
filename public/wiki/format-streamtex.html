Streaming (.stream / .streamtex)

Stream files are headerless payload blobs that hold the bulk pixel and geometry data the engine pages in on demand. Two siblings exist: .streamtex carries the full-resolution, RSX-swizzled texture pixels described by a companion .textures stub, and X.model.stream carries the vertex/index buffers for the matching .model. Neither has a magic or table of its own — all structure lives in the descriptor file that points into it. The eboot groups both under a "Stream Resources" loader, addressing them with the format strings %s|%s.streamtex, %sMD.model.stream and %s(noassert)|%s/%s/%s.model.stream.

Overview

Split/Second keeps a small always-resident descriptor next to a large streamed payload so the engine can display a low-res stand-in immediately and DMA the high-res data later. The pairing is:

.streamtex — texture payload

A .streamtex is purely the concatenated pixel data for the full-resolution textures listed in its companion .textures stub. It begins with no header — the first bytes are already swizzled DXT blocks:

000000  01 00 00 00 55 55 55 55 01 00 00 00 55 55 55 55   ....UUUU....UUUU
0001e0  ef 19 00 00 55 55 55 01 ef 19 00 00 55 55 05 00   ....UUU.....UU..
0001f0  ef 19 00 00 55 15 00 00 ef 19 00 00 55 00 00 00   ....U.......U...
000200  ef 19 00 00 54 00 00 00 ef 19 00 00 55 50 00 00   ....T.......UP..

Annotation: these are exactly the byte patterns that appear inline inside a non-streamed .textures file's payload region — the 55 55 55 55 / ef 19 00 00 sequences are the RSX swizzle of large flat-colour DXT1 fields. The .streamtex is simply that payload region lifted into its own file so it can be streamed separately.

Layout is dictated by the .textures stub

The stub's descriptors give, in order, each full-resolution texture's gcmFormat, width, height, mip count and dataOff into the stream. For Musclecar_01 the full-res set is:

TextureFormatSizeDXT1 + mips
paraboloid_back / paraboloid_frontDXT1512×5122 × 174,776 B
Musclecar_01 / Musclecar_01_DamageMapDXT11024×20482 × 1,398,120 B
DirtMap / ScratchMap / MetalMapDXT1256×2563 × 43,704 B
Summing those full-resolution mip chains gives 3,276,904 bytes against the actual Musclecar_01.streamtex size of 3,277,312 bytes — a 408-byte difference attributable to per-texture block alignment. This size identity is the primary evidence that .streamtex is the full-res pixel payload for its .textures twin.

.model.stream — geometry payload

The companion to a .model, this blob holds the interleaved vertex stream and index data. It is likewise headerless — the .model file supplies the buffer descriptors (stride, attribute layout, counts) that interpret it. The first bytes are packed vertex components (half-floats / normalized shorts), not ASCII or a magic:

000000  00 00 00 00 00 12 ad ae 00 00 00 00 38 32 3a 6d   ............82:m
000010  bf bc 3c 00 33 63 1e fe bb c9 35 58 38 36 3a 97   ..<.3c....5X86:.
000020  bf bb 3c 00 33 5b 23 35 bb c9 33 88 37 d3 3a 6c   ..<.3[#5..3.7.:l
000030  bf c4 3c 00 32 b9 20 62 bb d2 35 34 37 d4 3a 97   ..<.2. b..547.:.

Annotation: after a 12-byte leading block (a zero word, a small value 00 12 ad ae, then a zero word) the data settles into a fixed 16-byte stride: each row carries a recurring 3c 00 and the same trio of high bytes (bf bb/bc/c4, bb c9/d2, 3a 6c/6d/97) — the signature of an interleaved vertex buffer of packed half-float / normalized-short positions and normals. Decoding the exact attribute layout requires the .model header and is covered on the Model page; here we only establish that .model.stream is raw geometry, not another TEXS container.

The .model.stream internal vertex format is Partial: the stride is regular and clearly geometry, but per-attribute semantics belong to the .model descriptor and are not re-derived here.

The Stream Resources loader

The eboot string table contains a "Stream Resources" heading immediately followed by the resource-path templates used to locate these blobs:

Low LOD Geom
L.model
High LOD Geom
Sub Sectors
Load Sectors

…
Stream Resources
%s|%s.streamtex
.Static.ark
…
%s(noassert)|%s/%s/%s.model.stream
%s(noassert)|%s/%s/%s.streamtex

The %sMD.model.stream template sits in the sector-streaming block next to the "Low/High LOD Geom" loaders; the %s|%s.streamtex template heads the "Stream Resources" block immediately followed by .Static.ark; and a pair of (noassert) path templates names both extensions together. The pipe (|) separates a residency/bucket tag from the path, the same convention used by the engine's Resident|… entries. The neighbouring .Static.ark reference indicates these streams are also packaged into .ark archives in shipping builds; in this loose devkit build they sit on disk next to their descriptors. See Resources & Streaming for the loader-thread / residency model.

How we know

  • No magic: first bytes of every .streamtex / .model.stream are payload data, not a 4-CC.
  • .streamtex = full-res pixels: its byte content matches the inline payload patterns of a non-streamed .textures, and its size matches the sum of the stub's full-resolution DXT mip chains.
  • .model.stream = geometry: regular 16-byte vertex-like stride of half-float exponents; paired 1:1 with a small .model descriptor.
  • Loader identity: eboot "Stream Resources" block names both extensions via %s.streamtex and %s.model.stream path templates.
