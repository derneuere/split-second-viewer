Nemesis & Helicopter Modes

Split/Second's "boss" content — the rampaging articulated Nemesis truck and the missile-firing attack helicopter — is implemented as a component-based AI entity called the Nemesis. The same component tree drives a ground truck, a demolition drone, or a flying chopper; a template file picks the Type and toggles which components run. This page documents that system from GameLogic/Nemesis/*.params, the cNemesis* classes in the eboot, and the five dedicated nem_* arenas, with real configured values.

The Nemesis component system

A Nemesis is an entity assembled from cNemesis* components, each tuned by a section of a template .params file. The eboot exposes the full class roster; the important ones are:

ComponentRole
cNemesisAirEngineComponentHelicopter flight model (velocity, wobble, wiggle, look/right-axis rotation).
cNemesisCarEngineComponentGround drive model for truck/demolition drone.
cNemesisKeyframedPhysicsComponentScripted/keyframed truck physics ('Vehicle filename' = NemesisTruck01).
cNemesisHelicopterAIComponentChopper brain: states, target distances, hover height.
cNemesisTruckAIComponentTruck brain: lane, drive speed, coordination group.
cNemesisDemolitionAIComponentDemolition drone skill ramp.
cNemesisThrowDebrisComponentBarrel / missile launcher with pooled ordnance.
cNemesisLevelComponentDifficulty "level" that escalates the assault.
cNemesisPlayerTriggerableComponentHealth and powerplay-damage values.
cNemesisTargetPlayerComponentWhich player to chase, and at what range.
cNemesisAggressionComponentTruck lane-blocking aggression.
cNemesisMissileTargetComponentThe lock-on reticle entity for chopper missiles.

Other classes seen in the eboot fill in the events and queries: cNemesisBattleAIComponent, cNemesisPowerplayComponent, cNemesisAudioComponent, cNemesisCarExplosionComponent, plus event types like cNemesisFrenzyEvent, cNemesisBlowUpEvent, cNemesisMissileHitHelicopterEvent and a family of cNemesisQuery* events (QueryHealth, QueryLevel, QueryWave, QueryMissileDamageChopperRatio, …).

The terminal section of each template sets the entity type, e.g. /GameLogic/Nemesis/TruckTemplate → Type = Truck; and /GameLogic/Nemesis/HelicopterTemplate → Type = Helicopter;. The boss game-mode files reference these templates from their /RaceSetup/Nemesis/Selection0…23 wave slots.

The Nemesis truck

The truck is the season's recurring articulated-lorry boss. Its handling comes from cNemesisKeyframedPhysicsComponent / cNemesisCarEngineComponent, while NemesisTruckPhysics.params defines the actual rigid bodies. Headline figures:

/GameLogic/Nemesis/NemesisTruckPhysics/BasicPhysics:
	'000;Mass' = 1500.0 (1.0, 3000.0);
	'001;TopSpeed(MPH)' = 120.0 (0.0, 200.0);
	'002;Max steer angle(deg)' = 7.0 (0.0, 35.0);
/GameLogic/Nemesis/NemesisTruckPhysics/Trailers/RigidBody:
	'000;Length' = 6.80;  '001;Width' = 3.0;  '002;Height' = 2.30;
	'003;Mass' = 50000.0 (0.0, 100000.0);
	'004;LinearDamping' = 9.0;  '005;AngularDamping' = 4.5;

So the cab is a light, twitchy 1,500 kg body limited to a 7° steer angle, dragging a 50-tonne trailer — the mass ratio is what produces the menacing jack-knife swing. The trailer is joined at Trailers/Pivots (truck pivot Y = 1.35, trailer pivot Z = −5.64) and acceleration falls off with a 10-point curve (AccelerationVsSpeed) and a steer-angle penalty table (AccelMultiplierVsSteerAngle).

Truck AI & aggression

/GameLogic/Nemesis/TruckTemplate/cNemesisTruckAIComponent:
	'Coordination group' = Truck;
	'Drive Speed' = 50.0 (0.0, 200.0);
	'Slowdown multiplier' = 0.8;  'Speedup multiplier' = 1.2;
/GameLogic/Nemesis/TruckTemplate/cNemesisAggressionComponent:
	Enabled = true;
	'Max lane change cap' = 2.0;  'Min lane change cap' = 0.0;
	'Per Level Blocking Increase' = 0.15;

Trucks belong to the Truck coordination group so multiple cabs cooperate; aggression (active lane-blocking) ramps up by 0.15 per difficulty level. The truck targets the player only inside a window (cNemesisTargetPlayerComponent: max range 150 m, 300 m in sudden death, "Backward Progress", "Closest player"), with a 70 m "No Target Zone" so it does not sit on top of you.

Throwing barrels & missiles

The truck's ordnance lives in cNemesisThrowDebrisComponent, organised into pools. The two main pools are the blue "AoE" barrels and the red instant-kill barrels; two more activate in Sudden Death and a fifth handles homing missiles:

PoolModelInitial → MaxInstant kill?Powerplay FX
1. AoEtruckbarrel_physics2 → 5noNemesis_BlueBarrel_Explosion_P
2. Deathtruckbarrel2_physics1 → 4yesNemesis_RedBarrel_Explosion_P
3. SD AoEtruckbarrel_physics0 → 0noNemesis_BlueBarrel_Explosion_P
4. SD Deathtruckbarrel2_physics6 → 12yesNemesis_RedBarrel_Explosion_P
5. Missile—0 → 0no(none set)

Barrels drop from offset (0, 3.125, −16.9) behind the cab with a 3 s ± 1.5 s fuse; the active mechanic is Sequential (cNemesisThrowDebrisComponent/Release Mechanics → 'Active Release Mechanic' = Sequential). The missile launcher (ReleaseMechanicMissile) lobs 6 (easiest) → 12 (hardest) projectiles, reaching hardest difficulty after 5 levels, with a 1.25 s aiming state and a 0.6 s lock — the same mechanic the chopper reuses. In TruckTemplate.params the missile pool itself ships dormant (Initial → Max = 0 → 0) and has no powerplay FX assigned; the Nemesis_Missile_Explosion_P / Nemesis_Missile_Trail_P effects are configured only on the live chopper instances (see The attack helicopter below).

The attack helicopter

The chopper is a Bell 206B ('Visual model filename' = 'Generic\Models\Helicopter_Bell206B_01\Helicopter_Bell206B_01') flown by cNemesisAirEngineComponent + cNemesisHelicopterAIComponent. The flight model is deliberately wobbly:

/GameLogic/Nemesis/HelicopterTemplate/cNemesisAirEngineComponent/Movement/Velocity:
	'Max Velocity' = 130.0 (10.0, 200.0);
	'Time To Get To Target' = 0.4;  'Velocity Bias' = 2.90;
/GameLogic/Nemesis/HelicopterTemplate/cNemesisAirEngineComponent/Movement/Wobble:
	'Amplitude X' = 2.0;  'Amplitude Y' = 0.5;  Frequency = 5.0;
/GameLogic/Nemesis/HelicopterTemplate/cNemesisAirEngineComponent/Movement/Acceleration:
	'Max Acceleration' = 30.0 (0.0, 30.0);

Helicopter AI states

The chopper brain is a small state machine; each state sets how far it tries to sit from its target and how cautiously it corners:

StateMin → Max target distCornering cautionNotes
Find Race Leader35 → 85 m3.0"Ahead Time" 4.0 s — flies to intercept the leader.
Triggerable40 → 70 m1.5Waiting to be set off by a powerplay; start height 60.
Raining Death75 → 100 m3.0Active bombardment of the lead pack.
Fly Away——Retreat multiplier 30.
Waiting——Re-engages once the player is within 100 m.

Hover behaviour comes from the Height block: it holds 20 m over the track, looks 60 m ahead to anticipate terrain, with a 100 m "Trigger Height". Barrel drops are gated on Barrel Conditions — drop 50 barrels, only when "At Target". The chopper uses the Helicopter release mechanic (vs. the truck's Sequential), confirmed in both live configs: cNemesisThrowDebrisComponent/Release Mechanics → 'Active Release Mechanic' = Helicopter;

Helicopter health & powerplay damage

Both live chopper files share the same vulnerability values from cNemesisPlayerTriggerableComponent:

/GameLogic/Nemesis/HelicopterRevenge/cNemesisPlayerTriggerableComponent:
	DamageForLevelOnePowerplay = 1 (0, 128);
	DamageForLevelTwoPowerplay = 4 (0, 128);
	HelicopterLife = 16 (0, 128);

So the chopper has 16 hit points: a Level-1 powerplay shaves 1, a Level-2 powerplay 4. Both Revenge and Survival enable a missile-target reticle (cNemesisMissileTargetComponent, 'Target model filename' = Generic\Models\Missile_Target\Missile_Target) so the player can shoot back.

Revenge vs. Survival — the level escalation

The difference between the two helicopter modes is entirely in cNemesisLevelComponent, which decides what makes the assault harder. 0xDEADBEEF (-559038737) appears as the uninitialised value for "Levels To Lose When Player Wrecked" in both — i.e. that sub-rule is unset.

Level driverHelicopterRevengeHelicopterSurvival
Nemesis DestroyedEnabled (+1 / kill)Enabled (+1 / kill)
Nemesis LifeEnableddisabled
Timedisabled (10 s/level)Enabled, 9 s/level, ×0.95 per level
LinearTimedisabled (max 20, 180 s)disabled (max 20, 180 s)
Helicopter Wavedisabled, 3 waves/leveldisabled, 6 waves/level
Trucks Overtakedisableddisabled

In short: Revenge escalates by killing choppers and tracks chopper life (a directed "boss fight"), while Survival escalates on a shrinking 9-second timer that gets 5% shorter every level — an endless endurance run. Both end on the Lives end condition with a 60 s initial countdown, and report scoring as Chopper Revenge / Chopper Survival respectively.

The truck template's cNemesisLevelComponent instead enables Trucks Overtake (+1 level each time the player overtakes a truck) — the boss-truck mode escalates the more you push past the convoy.

Demolition & Tutorial drones

The two tiny templates reuse the Nemesis system as ordinary Demolition-type drone cars built on Musclecar_Drone. The only difference is the skill ramp on cNemesisDemolitionAIComponent:

The tutorial drone is a sitting duck (skill 0, vulnerable to AoE) so new players can practise pushing it into powerplays; the demolition drone is a maxed-out target.

The nem_* arenas

Boss modes run on five dedicated, looping arenas listed in directory.txt (distinct from the main campaign levels). Each has its own Subtracks/* set and is tagged :nem_name_SUBTRACK for the loader:

ArenaTheme
nem_downtownCity streets boss arena.
nem_graveyardCemetery / industrial outskirts.
nem_stormStorm-drain / aqueduct arena.
nem_warehouseWarehouse district.
nem_trainingTraining arena (tutorial/practice).

The same section of directory.txt also lists the boss vehicles loaded for these modes: NemesisTruck01 and Truck_Demolition. The Tracks & Levels page covers the subtrack/route pipeline these arenas use.
