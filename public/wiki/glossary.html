Glossary & Codenames

The Black Rock engine is full of internal codenames and acronyms. This page defines them, each anchored to a real string from the game data or EBOOT.BIN so you can trust the definition rather than guess. Terms are grouped by area; cross-links point to the page that documents each one in depth. For the big picture start at Engine Architecture.

Engine & rendering

Crayon2

Black Rock's render-support library and the renderer's codename. Source path libs/Render/Support/Crayon2; runtime init symbol Crayon2_Init; shaders are runtime-patched in Crayon2/Shader/PS3/ShaderPatching.cpp. Drives the deferred G-Buffer pipeline. The "2" implies a successor to an earlier Crayon used on the studio's prior titles. Confirmed — see Rendering Pipeline.

G-Buffer

The geometry buffer of the deferred renderer: a set of multiple render targets written in the opaque pass and consumed by lighting. This build has five channels, each with its own debug toggle under Lighting/Debug/GBuffer/: albedo, unlit (emissive/pre-lit), normal, motion blur (velocity), specular. Stage labels include GBuffer End MRT, GBuffer SSAO, GBuffer Blur, GBuffer ZCull Restore. Confirmed.

SPULight

Deferred lighting executed as SPU jobs rather than on the RSX. Job labels: SPULight Build Jobs, SPULight Copy Depth, SPULight Stall Kick Jobs, SPULight Stall Global Light, SPULight Window of opportunity. Tunable from the debug menu under Lighting/LocalLights/SPULight/ (Enabled, Scale Bounding Sphere, Debug Render, Exclusive). The lighting result texture is deferred_lighting_SPULightTex. Confirmed — see Rendering Pipeline.

ZCull

The RSX hardware depth-cull feature, used by Crayon2 to reject occluded pixels early. Appears as the frame stage GBuffer ZCull Restore. Confirmed.

JobStream

Black Rock's job/task system, layered over Sony's SPURS SPU runtime. Source path libs/System/JobStream/Spurs (SpursJobChain, SpursJobGuard, SpursJobEventHandler, cJobThread). The EBOOT contains the literal chain-init call cellSpursJobChainAttributeInitialize(…, 256, 1). Confirmed — see Threading & JobStream.

SPURS

Sony's SPU Runtime System — the PS3 OS facility for scheduling work across the Cell's SPUs. Not Black Rock's; it is the substrate JobStream sits on. Confirmed.

Edge

Sony's Edge SPU library suite (contrib/Sony/Edge). Used here mainly as EdgeGeom for geometry decompression/processing on SPU, wrapped by the engine's libs/Render/Model/EdgeModel (e.g. EdgeCustomTypes_spu). Confirmed — see Middleware Stack.

Gameplay data & FX

Catnip

The data-driven entity / timeline / scripting system that choreographs gameplay and FX. EBOOT paths reveal its breadth: Catnip/Trigger/, Catnip/PowerplayTimelines/, Catnip/Powerplay/Timeline/, Catnip/Physics/InteractiveObjects/, Catnip/Audio/Timeline/AudioEvent/, Catnip/Audio/{Trigger,Portal,SwooshBox}/, Catnip/LensFlare/, Catnip/Lighting/. It is the glue between triggers, interactive objects, audio events, lights and the Powerplay sequences. Confirmed — see Catnip Entity System.

Powerplay

Split/Second's signature mechanic: spending a charged power bar to trigger track-altering events ("powerplays") against rivals. One of the most heavily referenced gameplay systems in the binary (Powerplay ≈ 226 string hits). Implemented by cPowerplayManager, cPowerplayController, cPowerplayMaterialManager and PowerplaySequenceDirector, choreographed by Catnip timelines and a dynamic "danger zone" trigger system (GameplayParameters/Powerplays/Dynamic Trigger, Extra extent to add on the danger zones). Confirmed — see Powerplay System.

Megabowles

An automated Powerplay-scenario test/profiler harness (class cMegabowlesController; Megabowles ≈14 string hits). Driven by data under MegabowlesScenarios/ with filters such as Filter/powerplay_name, Filter/powerplay_name_at_start, Filter/sector_name, plus RenderScenarios and repeat_scenarios. Used in QA to mass-fire powerplays and catch crashes/regressions. The name is studio in-joke flavour (cf. cricketer Bob Bowles). Confirmed — exposed because qa_menus = true (see Build & File Layout).

AI

SSAI

The game's AI system — "Split/Second AI". Class/data prefix SSAI (≈161 hits); data root Deferred/SSAI/. Covers driving behaviour (via LDC tables), the AI power bar, per-car info (SSAI/PerCarInfo.params), race scripting (SSAI/RaceScript/) and the Evolution test harness. Confirmed — see AI (SSAI).

LDC

The per-speed AI driving tables. Stored as Deferred/SSAI/AI{Class}_{NN}_LDC.params and a Default_LDC.params. Each file defines three speed bands — 60, 120 and 180 — with the same 20 tuning fields per band (steering max angle, braking-onset angle, throttle/brake multipliers, track look-ahead distance, lane compensation, drift angles/look-ahead, and Easy/Medium/Hard hazard look-ahead & speed multipliers). The speed bands come from GameplayParameters/LDC: Initial Speed = 60, Final Speed = 180, Speed Intervals = 60. Example real value: '004;Distance to look ahead on the track for steering' = 40.0 (5.0, 100.0); at Speed 120. The expansion of "LDC" is not spelled out in the data; the contents are a per-speed driving-control table. Partial (contents Confirmed, acronym expansion unconfirmed) — see AI (SSAI).

Evolution (EvolutionConfig)

An automated AI fitness-test harness. Deferred/SSAI/EvolutionConfig.params defines per-variable test sets (e.g. Drift/Curvature ahead factor, Drift/Sideways factor), each with up to 8 TestN cases specifying a start position, a target "final zone" (centre + radius) and a time limit, plus Number of variables to try in the range. The AI is respawned at the start and scored on reaching the zone — a search over driving parameters. Confirmed.

Power bar (AI Powerbar)

The charge meter that gates powerplays, for AI as well as the player. GameplayParameters/AI Powerbar/ defines exactly how AI fills it: Drafting (min distance 5.0, max angle 5°, 4.0 units/sec), Drifting (14.0 units/sec), Drive Fast (above 60% of max speed, 1.0/sec), Jump (after 0.5 s airborne, 10.0/sec). Related debug: Powerbar Layer, Enable Powerbar Component, Throw Only Death Barrels With Full Powerbar, AIStat_CurrentPowerbar. Confirmed.

DCB

The AI difficulty / rubber-band balance factor. Built from a player term plus modifiers: Player DCB, Event DCB Modifier, Track DCB Modifier, Vehicle DCB Modifier (each car in VehicleDatabase.xml carries a DCBModifier attribute, e.g. Musclecar_07 = 06). Tuned under RaceSetup/DCB/AI Skill/ (Initial/Final Range Min/Max Limit) and surfaced on a Season Mode DCB HUD. Functions as the rubber-band/catch-up skill scalar. The acronym's exact expansion is not stated in the data (it behaves as a "dynamic competitive balance" value). Partial (mechanism Confirmed, expansion unconfirmed) — see Difficulty & Race Director.

Catchup

The torque-based rubber-banding applied to vehicles, separate from LDC driving. Configured in GameLogic/Catchup.params with per-class blocks like Prototype/Catchup/{Distance from back for full penalty, Max torque loss, Max torque gain, Top speed gain}. Confirmed — see Difficulty & Race Director.

Respot

The system that re-places ("respots") a crashed or off-route car back onto the track — central to keeping Split/Second's destructive races flowing. Class cRespotController (debug log InGame::cRespotController::DoUpdate - player %i respot); Respot ≈122 string hits. Heavily parameterised: Respot/{RespotPos X/Y/Z, RespotLook X/Y/Z, RespotSpeed, SpreadDistanceWidth/Length, RespotNumPlayers, Force Respot} and GameplayParameters/Respotting/ (max distance on correct route 500.0, max delay 3.0 s). Closely related is Teleporting (GameplayParameters/Teleporting/) for closed/scripted-route recovery. Confirmed — see Difficulty & Race Director.

Networking

ATVNet

Black Rock's in-house networking wrapper (the name carried over from the studio's ATV title Pure). Source path games/SplitSecond/Shared/ATVNet/NetZ; ≈137 hits. It wraps Quazal NetZ and implements replication via Duplicated Objects — the cATVNetDO* class family: cATVNetDOGameModeUpdater, cATVNetDOGameSetup, cATVNetDOPeerGameState, cATVNetDOPlayerSetup, cATVNetDOPlayerUpdater, cATVNetDOPhysicsObjectUpdater, cATVNetDORoomState. Confirmed — see Networking & Online.

Quazal (NetZ / Rendez-Vous)

The third-party networking middleware (contrib/Quazal/NetZ; Quazal ≈22 string hits). NetZ provides the Duplicated-Object replication model; Rendez-Vous (RV) provides matchmaking/lobby services. Voice chat rides Quazal's AVStreams with LPC10 and OpenLPC codecs (contrib/Quazal/NetZ/lib/Extensions/AVStreams/Codecs/{LPC10,OpenLPC}, VoiceChannel). Confirmed.

Audio

LLAudio2

Black Rock's audio wrapper around FMOD Ex 4.28 (libs/Audio/Wrappers/LLAudio2/…/FMOD; LLAudio2 ≈32 string hits). The FMOD library itself is the 4.28 branch, built in a separate ss_audio_dev workspace (contrib/FmodEx/42800/…). Subsystems include ReverbManager / DirectionalReverbManager, EventManager, SoundBankManager, GroupManager, AudioLoader and a PS3 init path (Init_PS3). Confirmed — see Audio Engine.

DynamicMixer

The runtime audio mixing controller (libs/Audio/Controllers/DynamicMixer; ≈59 hits). Uses preset/parameter/group stacks (PresetStack, ParameterStack, GroupPreset, GroupModifier, GroupPresetSlot) to blend mixer states by game context. Confirmed — see Audio Engine.

QA, telemetry & build

TrackTivity (TTY)

The QA telemetry recorder. Enabled by enable_TrackTivity = true in SplitSecond.params; writes .track capture files (this build left 111 in the data root because force_save_TTY_to_Devkit_HDD_root = true). "TTY" is the related developer console/log stream (save_TTY_logs_to_file, tty_streams_force_on). Confirmed — see TrackTivity (.track).

qa_menus

The master switch for in-game debug/QA menus (qa_menus = true). With it on, the engine exposes the deep debug trees referenced throughout this wiki (e.g. Lighting/Debug/GBuffer/, Respot/, RaceSetup/DCB/, DebugPages/AI/, MegabowlesScenarios/). Confirmed.

File formats & data

.params

The plaintext tuning format used pervasively. A !directory: header names the logical root, /path/section: lines open groups, and value lines read 'NNN;Human description' = value (min, max); where (min, max) is the editor's slider range. Booleans omit the range. Confirmed — see .params Tuning System.

.ark

The engine's packed asset-archive format (referenced even though this build is loose). Named set: Resident.ark, Frontend.ark, InGame1/2/3.ark, FrontendPowerplays.ark, per-level Environments/Levels/%s.ark and the %s.Static.ark / %s.Stream.ark split. Confirmed — see Archives (.ark).

Catnip file formats

The on-disk forms of Catnip data: .entities, .powerplays, .triggers, .timelineInfo, .logicinfo, .highlighttags.

EBOOT.BIN / SELF / SFO

EBOOT.BIN is the PS3 executable, an SCE SELF (Signed ELF) — its header magic is 53 43 45 00 ("SCE\0"). PARAM.SFO is the title metadata table (magic \0PSF) holding TITLE_ID, CATEGORY, etc. Confirmed — see Build & File Layout.

Studio & tooling lineage

The studio is Black Rock Studio, published by Disney Interactive. The source tree in the EBOOT roots at c:/Projects/proj_1/ws_0/Code/, split into libs/ (engine), games/SplitSecond/ (title) and contrib/ (middleware). The ATVNet codename (games/SplitSecond/Shared/ATVNet/) carries over from the studio's earlier ATV racer Pure. oscpack (contrib/oscpack/oscpack/{osc,ip/ps3}) is an OpenSound Control implementation over UDP, almost certainly used by the audio/tools pipeline rather than gameplay. (Black Rock was historically the former Climax Brighton / "Climax Racing" studio — a real-world fact, not visible in this build's data.) Confirmed (paths) / Theory (oscpack purpose).
