Bink Video (.bik)

Every full-motion video in Split/Second — the studio idents, the season-episode "Previously on…/Next time" recaps, the animated frontend backdrops and the per-mode tutorial clips — is a RAD Game Tools Bink file. All 29 movies live in Deferred/Movies/ and use the classic BIKi container with a self-describing fixed header followed by a per-frame offset table. The engine wraps the Bink runtime in the in-house MMBinkMovie class.

Identification & magic

The first four bytes are the ASCII signature BIK followed by a single codec/revision letter. Every Split/Second movie uses BIKi (42 49 4B 69) — the "i" generation of the Bink 1 codec (Bink 2, KB2*, is not used here). The codec letter is constant across all 28 samples examined.

Despite the PS3 being a big-endian PowerPC platform, the Bink container is little-endian: this is RAD's own cross-platform format and the runtime byte-swaps internally. Every multi-byte field below reads as little-endian — verify by checking that file_size + 8 == on-disk size.

Header layout

The header is a fixed 44-byte block (0x00–0x2B) up to and including num_audio_tracks. When num_audio_tracks > 0, a per-track audio descriptor block follows; then comes the frame-offset table (num_frames + 1 entries) and the interleaved audio/video packets. The fields below were decoded by cross-comparing a small clip (Blackrock.bik, 11.7 MB) against a 640×360 montage (DIS.bik) and several 720p and 256×256 clips; the resolution, frame-rate and audio-track values change exactly where the visible content differs, while the magic and reserved bytes stay constant.

The low bit of each offset is a keyframe flag — mask with ~1 to seek. Confirmed (the first masked entry lands exactly on the byte after the table).

Frame-rate field

The frame rate is stored as a rational, not a float, which is how RAD encodes both integer and NTSC rates without rounding. Split/Second uses five distinct rationals across the 29 movies (verified by reading 0x1C/0x20 on every file):

Audio-track count

Field 0x28 is a direct count of multiplexed audio streams. It tracks the role of the clip exactly:

The silent SS_BG_MaiN_f2_720_unc.bik (the animated main-menu background) and the small 256×256 FE_* backdrops all carry zero audio tracks, a clean cross-check that field 0x28 really is the audio-track count and not, say, a colour-space flag. Note the frontend loops (FE_*, SS_Ident) are 256×256 surfaces, not full-screen — they are textured onto Scaleform menu panels rather than played at native resolution.

How the engine loads them

Movies are opened through the MMBinkMovie wrapper and a dedicated I/O thread. The EBOOT carries the matching debug strings:

BinkLoad · Opened Bink movie %s · Failed to open Bink movie (%s): %s Trying %d more times. · Setting Bink Audio with %d channels · Bink IO Thread · Bink Audio Thread · Not a Bink file. · Error reading Bink header.

The retry-on-open string and a separate I/O thread are consistent with movies being streamed from disc rather than fully resident. Many entries in the residency list are tagged Resident|Movies/… (e.g. Resident|Movies/FE_MM_Static.bik), meaning small frontend loops are pinned in memory while the long episode recaps stream. The Scaleform frontend triggers playback through ActionScript hooks (_root.StartMovie / _root.StopMovie, "Movie Player", "InGame Movie", Intro_NumMovies, Intro_Bink), so the same Bink files double as both fullscreen cutscenes and textured surfaces inside Scaleform GFx menus.

The movie set

The 29 files split into clear buckets: studio/publisher idents (Blackrock, DIS, DISBR, SS_Ident), 256×256 animated frontend backdrops (FE_MM_0103, FE_MM_Static, FE_Online, FE_Options, FE_Quickplay, FE_SplitScreen) plus the 720p main-menu loop SS_BG_MaiN_f2_720_unc, per-mode tutorials (six Tutorial* clips), season-show framing (PreviouslyOnSS, NextTime, EpisodeTeaser, SeasonFinale, SeasonModeTutorial), a vertical-slice promo (SS_VSlice_Xmovie_02, 133 MB), an unlock sting (Car_Unlock_FC_UNC) and generic placeholders (Vid1–Vid3). Sizes range from the 2.16 MB Vid3 up to the 155 MB SeasonFinale. Files with an _unc/_UNC suffix (e.g. SS_BG_MaiN_f2_720_unc, Car_Unlock_FC_UNC) appear to be uncompressed-audio / higher-bitrate masters.

Confidence

Confirmed for the magic, file_size, frame count, resolution, frame-rate rational, audio-track count, the per-track audio descriptor block (max-packet / sample-rate+flags / id), the frame-offset table and the low-bit keyframe flag — all verified by reading every one of the 29 files and by parsing Blackrock's and TitleSequence's audio block and offset table to the EOF marker (the masked first frame offset lands exactly on the byte after the table). Opaque for the compressed video/audio bitstream itself — that is RAD proprietary Bink data, decoded only by the licensed runtime. The exact bit meaning of the 0x7000 audio-flags nibble (stereo/codec) is inferred from RAD's published Bink layout rather than measured here.
