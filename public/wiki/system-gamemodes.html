Game Modes

Split/Second is built on a single, enormous race-setup schema. Every game mode — Race, Elimination, Survival, Detonator, and the boss modes — is the same /RaceSetup registry tree with different switches flipped. There is no separate "mode" code path: a mode is just a .params file under GameLogic/GameModes/ that enables one of a handful of "End Conditions", picks a scoring profile, and tunes the power-bar / scoring economy. This page documents each shipped mode with the real configured values read from the NPXX00575 devkit build.

How a mode is defined

Each mode file opens with !directory: /RaceSetup and then sets dozens of grouped sections. The same skeleton appears in every file; what differs is which switches are true. The most important groups are:

  • /RaceSetup/Cameras → 'Game Mode Name' — the internal mode tag (e.g. Elimination, Demolition, TimeTrial, Helicopter).
  • /RaceSetup/AI → 'Difficulty parameters file' — which AI difficulty profile to load (Elimination, Demolition, Champ, '001Default' …), handed to the SSAI system.
  • /RaceSetup/Logic/EndGame — a set of mutually-exclusive End Condition booleans (Laps, Elimination, Countdown, Lives, Wrecks, Last x Num Player, Nemesis Destroyed, Tug-Of-War) that decide when the race terminates.
  • /RaceSetup/Scoring → 'Scoring GameMode Used' — the scoring profile name reported to scoring / telemetry.
  • /RaceSetup/Powerbar and /RaceSetup/Scoring/* — the power and points economy.
  • /RaceSetup/Nemesis — for boss modes, the wave roster (Selection0…Selection23) referencing Nemesis templates.
The eboot exposes one localisation key per mode: TEXT_GAMEMODE_NAME_RACE, _ELIMINATION, _DEMOLITION, _TIMETRIAL, _HELICOPTER, _HELICOPTERMULTI, _NEMESIS, _NEMESISMULTI (plus the generic TEXT_GAMEMODE_NAME_%s / _%s_HELP). These are the eight player-facing modes; the other .params files are setup variants (Quick Race, Champ, Car Capture, Learning, Tutorial, Development).

"Learning" files are AI-recording sessions, not playable modes: /RaceSetup/LDC → IsLearning = true and IsRecording = true, with 'Number of Laps' = 2048 so the AI laps forever while the LDC driving tables are captured. Every other mode keeps both false.

The End-Game switchboard

The end of a race is decided by /RaceSetup/Logic/EndGame. Exactly one of the End Condition * booleans is normally true. This is the single most useful field for telling modes apart:

End ConditionUsed byMeaning
End Condition LapsRace, Champ, Demolition, Detonator, Car CaptureRace ends after N laps (see /RaceSetup/Logic/Laps).
End Condition EliminationEliminationLast-placed car is dropped on a timer; also sets End Condition Last x Num Player = true.
End Condition LivesHelicopter (Revenge/Survival), NemesisRace ends when the player runs out of lives.
End Condition Countdown(available, off by default)Race ends when a shared countdown hits zero.
End Condition Wrecks(available, off by default)Race ends after N wrecks (Number_of_wrecks).
End Condition Tug-Of-WarDemolitionScore-based tug-of-war between teams/players.
End Condition Nemesis DestroyedHelicopter (Revenge/Survival)Race ends when the boss vehicle is killed; both chopper modes set this true alongside Lives.
End Condition Last x Num PlayerEliminationEnds when 'Num Players To End Race' remain.

Race / Quick Race

QuickRace.params is the canonical "everything on, nothing special" profile and a good baseline for the schema. Three laps, eight players, full powerplay system, scoring profile reported as Time Trial.

/RaceSetup/AI:
	'Difficulty parameters file' = '001Default';
/RaceSetup/FinishCondition:
	Number_of_laps = 3 (0, 128);
/RaceSetup/Logic/Laps:
	Enabled = true;
	'Number of Laps' = 3 (1, 2048);
/RaceSetup/Players:
	'Max players' = 8 (1, 8);
/RaceSetup/Powerbar:
	'Power required to activate level 1 (percentage)' = 33 (0, 100);
	'Power required to activate level 2(percentage)' = 66 (0, 100);
	'Power required to activate level 3(percentage)' = 100 (0, 100);

The three powerplay levels unlock at 33% / 66% / 100% of the power bar — these thresholds are shared by virtually every mode. The Race profile fires the four "GameEvent" powerplays on intro / start / lap / end (PA13_StartGloryMovie_P, PGE_Start_P, PGE_Lap_P, PGE_End_P).

Champ Race

ChampRace.params is the season/championship profile. It is identical to Quick Race apart from the difficulty file (Champ) and a tighter starting AI-skill band:

/RaceSetup/AI:
	'Difficulty parameters file' = Champ;
/RaceSetup/DCB/AI Skill:
	'01;Initial Range Min Limit' = 0.4 (0.0, 1.0);
	'02;Initial Range Max Limit' = 0.4 (0.0, 1.0);
	'03;Final Range Min Limit' = 0.1 (0.0, 1.0);
	'04;Final Range Max Limit' = 0.2 (0.0, 1.0);

It also tweaks the overall-rank ladder (/RaceSetup/Logic/Ranking): Rank 0 = Disqualified, Rank 1 = Distance Raced, Rank 2 = Total Race Time.

Elimination

The back-of-the-pack car is eliminated on a recurring timer. Elimination.params turns on the time-based eliminator and the danger-state HUD that drives the on-screen countdown:

/RaceSetup/Logic/Elimination/Time Based:
	'Elimination Time 1' = 60.0 (0.0, 180.0);
	'Elimination Time 2' = 20.0 (0.0, 180.0);
	'Elimination Time 3' = 20.0 ... 'Elimination Time 7' = 20.0;
	Enabled = true;
/RaceSetup/Logic/DangerState:
	Enabled = true;
	'Use EliminationTimer' = true;
/RaceSetup/Logic/EndGame:
	'End Condition Elimination' = true;
	'End Condition Last x Num Player' = true;

So the first elimination fires after 60 s, then every 20 s thereafter. The HUD lines are pre-wired to ELIMINATION_TARGET_POSITION and ELIMINATION_TIMER (both also appear as eboot format strings). A position-based eliminator exists too (Position Based: target 6th lap 1, 3rd lap 2, 1st lap 3) but ships disabled.

Detonator (TimeTrial)

The retail "Detonator" mode is internally TimeTrial: a single timed lap where the player detonates live powerplays against the clock. Its tell-tale switch is the forced single lap:

/RaceSetup/Cameras:
	'Game Mode Name' = TimeTrial;
/RaceSetup/Logic/Laps:
	'Number of Laps' = 1 (1, 5);
	'Override Default Number of Laps' = true;
/RaceSetup/Logic/DangerState:
	'Use EliminationTimer' = false;
/RaceSetup/Scoring:
	'Scoring GameMode Used' = 'Time Trial';

Because it is single-player vs. the environment, the elimination danger-state is off and the rolling start cruises at 130 mph (/RaceSetup/Logic/RollingStart → 'Cruise Speed' = 130.0). A RaceScript re-evaluation timer (30.0 s) re-assigns the powerplay scripts as the single car progresses.

Demolition

A score battle in which players push drone cars into powerplays for points. Demolition is the only stock mode that turns on the Tug-Of-War end condition and enables the score side of the economy:

/RaceSetup/Cameras:
	'Game Mode Name' = Demolition;
/RaceSetup/Logic/EndGame:
	'End Condition Laps' = true;
	'End Condition Tug-Of-War' = true;
/RaceSetup/Logic/Countdown:
	'Initial Countdown' = 180.0 (0.0, 3600.0);
/RaceSetup/Scoring/DemolitionWreck:
	'Base Value (Level 1 PP)' = 1000.0;
	'Base Value (Level 3 PP)' = 2000.0;
	'Enabled (Score)' = true;

It loads the Demolition AI difficulty file and uses the per-level respotting ladder under /RaceSetup/Logic/Respot to keep target cars in front of the player (the 24-step 'N Initial distance from racing line' table climbs from 150 m at level 0 to 2400 m at level 23); see the Nemesis page for the drone templates this respots.

Car Capture

Four near-identical 16,669-byte files (CarCapture1–4.params), one per car to be unlocked. They run the default difficulty profile and end on laps; their only meaningful per-file difference is the 'Game Mode Name' tag (CarCapture1…CarCapture4). These back the single-player "capture the rival's car" challenges.

Nemesis (boss truck)

Nemesis.params, NemesisMulti.params and NemesisSplitScreen.params set up the boss-truck mode. They end on Lives, load the Demolition AI difficulty profile and report scoring as Nemesis Trucks, and fill the 24 wave slots (/RaceSetup/Nemesis/Selection0…Selection23) with template references — the single-player file points each slot at TruckTemplate.params, the multiplayer file at TruckMultiplayer.params, and the split-screen file at TruckSplitScreen.params (unused trailing slots are None):

/RaceSetup/Nemesis:
	Selection0 = TruckTemplate.params;
	Selection1 = TruckTemplate.params;
	...
	Selection23 = None;           # 24 slots; unused tail = None

See Nemesis & Helicopter Modes for the truck/helicopter component trees and the wave/level mechanics.

The power-bar & scoring economy

Most of the difference between "fun" modes lives in /RaceSetup/Scoring/*, which controls how the power bar (and, when enabled, the score) is earned. The Quick Race values below are typical and are reused almost verbatim across modes:

ActionPower per secondNotes
Drafting2.0 → 4.0within 45 m, ±10° cone, ≥100 mph, ≥0.2 s
Drifting7.0 → 14.010°–90° slide angle; overtake bonus +8 power
Jumping5.0 → 10.0combos enabled; overtake bonus +8 power
Near Miss (environmental)10.0 (per event)≥70 mph, 0.1 s in box, 0.5 s cooldown
Nudge victim into crash66.67 (one-shot)/RaceSetup/Logic/PowerBar/NudgeVictim
Victim of a powerplay16.67award for being hit (disabled by default)

The AI earns power on the same actions but through a parallel table in GameLogic/GameplayParameters.params (AI Powerbar/Drafting|Drifting|Drive Fast|Jump) — documented on the AI (SSAI) page.

Many sub-systems exist in the schema but ship disabled in the stock modes — SelfFiringPowerPlay, TugOfWar (outside Demolition), Lives (outside boss/heli), the entire Lessons lesson-stage tree, and the PrototypeHud debug HUD. Their presence in every file is a schema artifact, not evidence the feature is active. Always check the Enabled flag of the specific sub-section.

Where modes are selected at runtime

directory.txt lists all 22 mode files under a :GAMEMODES tag, telling the loader to make them available as a set. The eboot resolves the player-facing names through TEXT_GAMEMODE_NAME_* localisation keys, and net play replicates the chosen setup via the cATVNetDOGameSetup / GameModeUpdater duplicated objects (see Networking & Online).
