Middleware Stack

Split/Second runs on Black Rock Studio's in-house engine, but — like every console title of its generation — it leans on a stack of licensed third-party libraries for physics, audio, networking, video, UI and SPU geometry. Every one of these libraries leaves source-tree paths and version banners behind in the EBOOT, so the exact versions are not guesswork: they are quoted verbatim below from eboot_strings.txt. All third-party code lives under c:/Projects/proj_1/ws_0/Code/contrib/<Vendor>/<Lib>/ in the build tree (the audio branch uses a parallel C:/Projects/SplitSecond/ss_audio_dev/Code/contrib/… root).

Physics — Havok 5.5.0

Every physics file in the data tree is a Havok 5.5 packfile (magic 57 E0 E0 57). The EBOOT is full of paths like contrib/Havok/Havok550/Animation/Animation/Animation/hkaSkeletalAnimation.inl, confirming that Black Rock used not only Havok Physics but also Havok Animation (hkaSkeletalAnimation) for car and character skinning. A dedicated Havok heap is allocated at startup — error strings warn when "we've blown the havok memory pool" and "Havok memory limit reached for geometry particle models".

The exporter pipeline tags physics objects with custom user-data containers whose class names survive in the binary, e.g. HavokTriggerPhantomExportUserDataContainer, HavokRespotBoxSystemExportUserData, HavokAreaOfEffectSystemExportUserData, HavokFurnitureObjectExportUserData and HavokAnimatedRigidBodyExportUserDataContainer. These map directly onto game systems: respot boxes (Difficulty & Race Director), trigger phantoms (triggers), and area-of-effect blast volumes.

There is a tuning constant for blending Havok's solver against a scripted line: CarCollision/004; Blend coefficient for mixing Havok response and on-rail behaviour. The cars are not pure ragdoll physics — they are a controlled mix of Havok response and an "on-rail" handling model. See Vehicle Physics (.phys).

Audio — FMOD Ex 4.28.00

The audio code was built in a separate branch (C:/Projects/SplitSecond/ss_audio_dev/Code/contrib/FmodEx/42800/…). The 42800 directory name is the FMOD version: 4.28.00. Both the low-level FMOD Ex API and the higher-level FMOD Event System are linked (error strings reference EventSystem::unloadFSB and FMOD_EVENT_INIT_FAIL_ON_MAXSTREAMS). The codec source files compiled in include FSB, Ogg Vorbis, MPEG, WAV/RIFF, AIFF, MIDI/DLS, IT/S3M/XM/MOD trackers and Sony SWVAG — though on disk Split/Second ships banks as .fsb with .fev event projects.

FMOD is never called directly by gameplay code; it sits under the in-house LLAudio2 wrapper (libs/Audio/Wrappers/LLAudio2). The full architecture — DynamicMixer, the EventManager/SoundBankManager pools, reverb, and the bespoke Tonic granular engine synthesizer — is documented on the Audio Engine page.

Networking — Quazal Net-Z 2008.9 SP2 + Rendez-Vous

The online stack is Quazal's middleware, the same family used across many late-2000s console titles. The banner string reads Quazal Net-Z / Version 2008.9 SP2 with Copyright (C) 1998-2009, Quazal Technologies Inc. Two Quazal products are present: Net-Z (object replication, the "Duplicated Object" / DO model) and Rendez-Vous (RV) (matchmaking and online services).

Black Rock wrapped Quazal in a game-side layer called ATVNet (games/SplitSecond/Shared/ATVNet/NetZ/). Each networked entity has a Duplicated-Object class, e.g. cATVNetDOGameModeUpdaterDDL, cATVNetDOPlayerUpdaterDDL, cATVNetDOPhysicsObjectUpdaterDDL and cATVNetDORoomStateDDL. Voice chat runs over Net-Z AVStreams using the LPC10 and OpenLPC codecs. A watchdog thread cKeepNetZAlive pumps the session. Full detail is on Networking & Online.

Video — RAD Bink

All pre-rendered video ships as .bik (RAD Bink). The wrapper class is MMBinkMovie and the loader logs Opened Bink movie %s / Setting Bink Audio with %d channels. There are dozens of Bink files — title sequence, "Previously on Split/Second" recaps, season finale, per-mode tutorials and frontend background loops (FE_MM_Static.bik, FE_Online.bik, etc.). No version banner is embedded, which is normal for Bink (the runtime is a static lib with no copyright string in the payload).

UI — Scaleform GFx 2.x

The frontend, menus and in-race HUD are built in Adobe Flash and rendered by Scaleform GFx. The loader expects the GFX 2.x file format — the error string is explicit: incompatible GFX file, version 2.x expected. The binary contains the full GFx object zoo (GFxLoader, GFxSprite, GFxDrawTextManager, GFxMorphCharacterDef, font/PNG/JPEG providers). GFx in turn relies on zlib (GFx_InflateWrapper) for compressed SWF/GFX data. UI assets are .gfx files; see HUD & Frontend.

SPU geometry — Sony Edge v0_4_3

Geometry processing on the PS3's SPUs uses Sony's Edge library. The path proves the version: contrib/Sony/Edge/EdgeGeom/spu/../../v0_4_3/target/spu/src/edge/geom/edgegeom.cpp (plus edgegeom_compress.cpp / edgegeom_decompress.cpp). Edge handles on-SPU vertex decompression and primitive culling that feeds the RSX, complementing Black Rock's own JobStream/SPURS jobs and the deferred renderer. Debug-menu hooks expose Renderer/Edge/LockEdgeCameraUpdate and Renderer/Edge/SelectBatchMin|Max.

Only EdgeGeom is positively identified in the strings. Edge also ships an animation library (EdgeAnim); whether Split/Second uses it (versus Havok Animation for skinning) is Theory — no EdgeAnim path was found.

Telemetry transport — oscpack

The oscpack library (Open Sound Control packet/UDP) is linked under contrib/oscpack/oscpack, with both the outbound packet stream (OscOutboundPacketStream.cpp) and the PS3 UDP socket (ip/ps3/UdpSocket.cpp) compiled in. OSC is a real-time messaging protocol popular in audio tooling; in this devkit build it is the most likely transport for live audio auditioning from the FMOD Designer host and possibly for TrackTivity telemetry streaming. The precise use is Theory — only the library's presence is confirmed.

How it fits together

A useful mental model of the dependency graph as actually linked:

  • Simulation → Havok 5.5 (physics + animation), blended with an on-rail handling model.
  • Rendering → Black Rock "Crayon2" deferred renderer, fed by Sony Edge on the SPUs.
  • Audio → LLAudio2 → FMOD Ex 4.28 (+ Event System) + the in-house Tonic synth.
  • Online → ATVNet → Quazal Net-Z 2008.9 (DO replication, voice) + Rendez-Vous (matchmaking).
  • UI → Scaleform GFx 2.x, decompressed via zlib, images via libpng/libjpeg.
  • Video → RAD Bink (MMBinkMovie).
  • Tooling/telemetry → oscpack (OSC/UDP) for live auditioning & data capture.
